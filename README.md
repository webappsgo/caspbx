# CASPBX

[![License](https://img.shields.io/github/license/casapps/caspbx)](LICENSE.md)

## About

`caspbx` is being built as a unified Asterisk communications platform that replaces the need for multiple separate PBX, operator, fax, end-user, call-center, and backend administration interfaces.

The repository is currently in the **foundation-building phase**. The codebase now contains the initial runtime, auth/session scaffolding, organization/custom-domain flows, a capability-driven Asterisk admin foundation, persisted PBX object foundations under `/admin/server/asterisk` and `/api/v1/admin/server/asterisk`, a live user communications foundation under `/users` and `/api/v1/users`, and an initial operator/call-center runtime foundation under the Asterisk admin tree.

## Features

- Research-backed product definition for a full Asterisk communications platform
- Minimum feature/capability parity target for the replacement surfaces
- Built-in, no-plugin product model with no feature gating
- Capability-driven UI policy so unusable features stay hidden until available
- Real auth, user, org, custom-domain, and admin/API runtime foundations
- Capability-aware Asterisk admin inventory for overview, health, modules, apply, media, fax, messaging, conference, queue, hardware, and browser-calling surfaces
- Persisted PBX object management for extensions, trunks, routes, queues, conferences, IVRs, prompt assignments, provisioning profiles, and apply-preview validation
- User communications control foundations for dashboard, contacts, call history, voicemail, messages, presence, webphone, and communications settings with capability-driven visibility
- Operator and call-center foundations for operator dashboard, trunk visibility, conference visibility, parked calls, queue wallboards, agent wallboards, and supervisor action previews
- Initial Go, Makefile, documentation, and container-testing bootstrap

## Production

Production deployment targets for `caspbx` are planned to include:

1. Container deployment under `docker/`
2. Native binary deployment for the `caspbx` server binary
3. Service-manager installation flows for long-running server operation

The runtime server implementation is not complete yet, so this repository should currently be treated as a **working platform foundation**, not a finished deployable PBX.

## Client

The project spec requires a companion `caspbx-cli` client and an optional `caspbx-agent`, but those binaries are not implemented yet in this bootstrap stage.

## Configuration

Configuration files are runtime-generated and are **not committed** to the repository.

Current implemented foundation status:

- `server.yml` is not committed and will be generated by the server on first run
- project paths, Docker layout, and documentation layout are scaffolded
- server config validation now includes custom-domain and Asterisk capability/admin foundation settings

## API

The live API foundation now includes health/version endpoints plus authenticated `/api/v1/auth`, `/api/v1/users`, `/api/v1/orgs`, `/api/v1/admin`, and `/api/v1/admin/server/asterisk` surfaces, including PBX entity/apply-preview endpoints, user communications endpoints for dashboard, contacts, call history, voicemail, messages, presence, webphone, and communications settings, and operator/call-center endpoints for operator dashboards, wallboards, and supervisor action previews.

The target platform replaces the practical need for multiple separate surfaces, including:

- PBX administration
- end-user communications portal
- operator and receptionist switchboard

- call-center reporting and live supervision
- system and backend operational administration
